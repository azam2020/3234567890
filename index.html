Issue Report: High CPU Utilization with Virtual Machines

Issue Summary

HSD No.: 3234567890

Title: Excessive CPU Utilization in Virtual Machines During Idle State

Description: Virtual machines (VMs) are showing high CPU utilization even when in an idle state. This abnormal behavior causes performance degradation across all VMs hosted on the same hypervisor.

Hardware Configuration

  • Hardware Model: Lenovo ThinkSystem SR650
  • CPU: Intel Xeon Gold 5218 (2x)
  • Memory: 256 GB RAM
  • Storage: 2x 960GB SSD, 2x 4TB HDD
  • Network Adapter: Broadcom BCM57416 10G Ethernet
  • Firmware Version: 4.8.0

Software Configuration

  • Operating System: VMware ESXi 7.0
  • Kernel Version: 5.4.0-92-generic
  • Virtualization Software: VMware vSphere 7.0
  • Hypervisor Settings: Default
  • VM OS: Ubuntu 20.04 LTS
  • VM Tools Version: open-vm-tools 11.2.5

Steps to Reproduce

  1. Install VMware ESXi on the Lenovo ThinkSystem server.
  2. Set up multiple virtual machines running Ubuntu 20.04 LTS.
  3. Ensure the VMs are in idle state (no active processes).
  4. Monitor the CPU utilization for each VM using the vSphere Performance tab or the command esxtop.
  5. Observe the CPU usage even in the absence of active workloads.

Expected Result

The CPU utilization should remain minimal during VM idle states, typically below 5%.

Actual Result

CPU utilization for the idle VMs ranges from 15% to 25%, which is unusually high for idle state.
